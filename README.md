# Personal Website

A minimal, zero-build personal website. Plain HTML/CSS/JS — no framework, no
build step, works on GitHub Pages out of the box.

## Editing content

All content lives in `data/` — you should never need to touch the HTML to
update the site:

| File | What it controls |
|---|---|
| `data/site.js` | Name, nav tabs, contact links, footer |
| `data/about.js` | Bio paragraphs, education, experience, skills |
| `data/courses.js` | Course reviews (one object per course) |
| `data/projects.js` | Projects (one object per project) |

Each file has a comment at the top describing its fields. To add a course
review or project, copy an existing object and edit it. Plain HTML is allowed
inside review/description strings (links, `<em>`, etc.).

Styling is in `css/style.css` — colors, fonts, and page width are CSS
variables at the top of the file. Page structure/markup is generated by
`js/render.js`.

## Adding a new page

1. Copy `projects.html` to `newpage.html` and change the `id` of the content
   `<div>` (e.g. `id="newpage-content"`) and the data `<script>` tag.
2. Create `data/newpage.js` with your content.
3. Add a render function in `js/render.js` (copy `renderProjects`) and call it
   from the `DOMContentLoaded` handler.
4. Add the page to `nav` in `data/site.js`.

## Previewing locally

Open `index.html` directly in a browser, or run a local server:

```sh
python3 -m http.server 8000
# then visit http://localhost:8000
```

## Deploying to GitHub Pages

1. Create a repo named `YOUR_USERNAME.github.io` on GitHub (or any repo name,
   which will serve at `YOUR_USERNAME.github.io/REPO_NAME`).
2. Push this directory to it:
   ```sh
   git remote add origin git@github.com:YOUR_USERNAME/YOUR_USERNAME.github.io.git
   git push -u origin master
   ```
3. In the repo's **Settings → Pages**, set the source to **Deploy from a
   branch**, branch `master`, folder `/ (root)`.

The site will be live at `https://YOUR_USERNAME.github.io` within a minute or
two of each push.
